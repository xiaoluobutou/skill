# 划线 → 洞见 (Highlight to Insight)

将阅读划线/摘录转化为**可复习、可回忆、可关联**的深度笔记。
核心理念：**输出式思考** —— 知识只有经过自己的大脑重新加工、建立连接，才能真正内化。

## 康奈尔笔记法简介

康奈尔笔记是美国康奈尔大学 Walter Pauk 教授提出的笔记系统，将一页纸分为四个区域：

```
┌──────────────────────────────────┐
│  标题：书名 · 作者 · 日期 · 章节    │
├────────────┬─────────────────────┤
│  线索栏     │  笔记栏              │
│  (Cue)     │  (Notes)            │
│            │                    │
│  · 关键词   │  · 从划线中提炼的    │
│  · 问题     │    核心知识点        │
│  · 提示     │  · 论据与例证       │
│  (1/3)     │  (2/3)             │
├────────────┴─────────────────────┤
│  总结栏 (Summary)                 │
│  · 用一两句话概括整页核心内容      │
└──────────────────────────────────┘
```

四个区域的作用：
- **标题栏**：书籍信息、章节、日期
- **线索栏**：提炼关键词、核心问题、记忆提示 —— 用于复习时自我测试
- **笔记栏**：基于划线整合的详细笔记
- **总结栏**：整页内容的一句话概括

## 工作流程

### 步骤 1：解析输入

首先识别用户提供的划线笔记来源格式：

**微信读书 Markdown 导出**（最常见）：
```markdown
# 书名
> 作者：作者名
◆ 章节名
>> 划线内容...
📝 想法：用户笔记...
```

**微信读书 JSON 导出**：
```json
{"bookName":"书名","author":"作者","notes":[{"chapter":"...","highlight":"...","note":"..."}]}
```

**Kindle My Clippings**：
```
==========
书名 (作者)
- 您在位置 #X-Y 的标注 | 添加于 YYYY-MM-DD
划线内容
==========
```

**纯文本**：用户直接粘贴的划线列表，自由格式。

如果格式无法识别，直接询问用户来源，或按纯文本处理。

### 步骤 2：内容预处理

在处理之前，对划线内容进行整理：

1. **去重**：移除完全重复的划线（同一句被多次划线）
2. **按章节分组**：如果原数据有章节信息，按章节组织；如果没有，按内容主题自动聚类
3. **区分划线与想法**：将原文划线与用户个人笔记/想法分开处理
   - 原文划线 → 进入「笔记栏」的素材
   - 用户想法 → 作为理解上下文，辅助 AI 提炼

### 步骤 3：AI 智能提炼

这是核心步骤。对每一章/每一组划线内容，进行三层提炼：

**第一层：关键词提取**
- 从划线中识别该章节的核心概念、术语、人名、方法论名称
- 每个章节提取 3-8 个关键词
- 关键词应该是「看到它就想起整段内容」的记忆触发器

**第二层：问题生成**
- 将核心知识点转化为「可测试的问题」
- 问题应该引导回忆，而非简单的是非题
- 好问题的特征：
  - ✅ 「为什么作者认为 X 是 Y 的关键原因？」
  - ❌ 「作者说了什么？」（太模糊）

**第三层：笔记整合**
- 将零散的划线整合为连贯的要点笔记
- 保留原文的关键论据、数据、例证
- 合并表达同一观点的多条划线
- 每条要点控制在 3-5 句以内，方便复习

### 步骤 4：撰写总结

为每个章节写一篇简短总结（3-5 句话）：
- 用自己（读者）的话重新表述，而非照搬原文
- 聚焦于「这一章的核心洞见是什么」
- 如果章节间有递进关系，在总结中标注

### 步骤 4.5：生成思维导图

在开始逐章笔记之前，先生成一个全书思维导图，给读者一个「鸟瞰视角」：

- 以书名为根节点
- 第一层：各章/各主题的核心论题
- 第二层：每个章节的 2-4 个关键概念
- 用树形结构呈现，每个节点控制在 15 字以内

### 步骤 4.6：映射边界知识

这是将书中的知识转化为**自己知识**的关键步骤。对每个章节的核心观点，寻找跨界关联：

**为什么要做边界映射？**
- 孤立的知识容易被遗忘，连接的知识形成网络
- 同一个原理在不同领域的表现，是理解的「试金石」
- 跨界联想让抽象概念变得具体可感

**如何寻找边界关联：**

1. **扫描关联域**：对每个章节的核心观点，逐一遍历以下领域——
   | 领域 | 思考角度 |
   |------|---------|
   | 💼 **工作/职场** | 这个原则如何应用到我的日常工作中？ |
   | 🏠 **生活/日常** | 日常生活中有没有类似的模式？ |
   | 👥 **社会/人际** | 这个观点如何解释社会现象或人际关系？ |
   | 🧘 **哲学/思维** | 背后有没有更底层的哲学或思维方式？ |
   | ⚖️ **道德/价值** | 涉及什么价值判断或道德困境？ |
   | 📜 **历史** | 历史上有没有相关的案例或教训？ |
   | 📚 **其他书籍** | 和哪些其他书的观点呼应或矛盾？ |
   | 🔬 **其他学科** | 生物学/物理学/经济学等有没有类似原理？ |

2. **筛选高价值关联**：不是每个领域都要有内容，只保留那些**真正有启发性**的关联。宁缺毋滥。

3. **具象化**：不要说「这个观点可以用在生活中」，而要说「这就像我们做菜时先备好所有食材（mise en place），再开始烹饪——减少中途手忙脚乱的可能」。

**边界知识的质量标准：**
- ✅ 「战术编程就像信用卡最低还款——每次只还最低额，利息却在不断累积」
- ✅ 「深模块的概念和产品设计中的'简单而强大'是同一个道理，比如 iPhone 只有一个按钮」
- ❌ 「这个观点很重要」（没有建立任何连接）
- ❌ 「这个观点和社会有关」（太模糊，等于没说）

```markdown

> **作者**：{作者} | **日期**：{日期} | **总划线**：{N} 条
```

### 模板规则

- **书籍级**：一份笔记对应一本书
- **章节级**：每个章节独立一页（重复「线索 → 笔记 → 总结 → 边界知识」区块）
- **思维导图**：在所有章节之前，提供鸟瞰视角
- **全书总览**：在所有章节之后，提供跨章节的汇总表 + 跨域知识地图
- 如果原文没有章节信息，按「主题聚类」代替章节，每个主题视为一个章节

## 处理原则

### 三大支柱缺一不可
- 🗺️ **思维导图**给出全局结构
- 🔑 **康奈尔笔记**用问题驱动理解
- 🌐 **边界知识**建立跨域连接，让知识"长"在已有认知树上

### 质量优先于数量
- 不是每条划线都要塞进笔记。AI 应判断划线的重要性，合并冗余，舍弃次要信息
- 宁可输出 5 条精炼要点，不要输出 20 条零散摘抄

### 理解优先于摘抄
- 笔记栏的内容应该是「理解后的重新表述」，而非原文照搬
- 如果划线和原文一模一样，说明提炼不到位
- 一个判断标准：读者看笔记时，能否不用翻原书就理解这个知识点？

### 问题驱动
- 线索栏的核心价值是「复习自测」—— 用户遮住右侧笔记栏，只看左侧问题，能否回忆起核心内容？
- 如果问题太模糊、太容易，或者不需要思考就能回答，就失去了价值

### 边界知识是知识内化的关键
- 孤立的知识点像漂浮的孤岛，边界知识把它们连成大陆
- 不要为了凑数而强行关联——如果某个章节确实没有好的跨界类比，可以留空
- 好的边界知识让读者产生"啊，原来如此！"的顿悟感
- 优先关联读者可能熟悉的领域（工作、生活），再拓展到更远的学科

### 保留原始出处
- 在笔记要点末尾，用 `[@第3章-划线12]` 格式标注出处
- 这让用户可以回溯到原始划线，验证理解是否准确

### 处理用户想法/笔记
- 微信读书导出中的「📝 想法」是用户自己的思考
- 将其融入笔记或总结中，用 `💭 我的思考：...` 标注
- 用户的想法往往是最有价值的个人洞见，不要丢失

## 边缘情况处理

### 无章节信息
按内容主题自动聚类，每个聚类作为一个「章节」。在聚类标题前标注 `[自动分组]`。

### 划线过少（<5 条）
如果整本书划线很少，合并所有内容为一页笔记，不强制分章节。
在总结中标注「本书划线较少，以下为整体笔记」。

### 划线过多（>100 条）
如果划线非常多，提供两种选项让用户选择：
1. **精选模式**：只取最有价值的 30-50 条进行整理，标注「已精选」
2. **完整模式**：逐章处理全部划线

默认使用精选模式，并在开头说明。

### 多本书混合
如果检测到输入包含多本书的内容（如微信读书导出了多本），
按书籍分开，每本书独立输出一份笔记。

### 非中文内容
保持原语言输出。如果输入是英文，笔记也用英文；中英混合则保持混合。

### 用户反馈循环
如果用户在看完输出后说「这部分不对」「这个词理解错了」，
先更新对应部分，再询问是否需要重新生成整份笔记。
